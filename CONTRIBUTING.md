# Contributing Guide

Thank you for your interest in the Awesome AI Waifu project! We welcome all forms of contributions.

[中文文档](CONTRIBUTING.zh-CN.md)

## How to Contribute

### Adding New Resources

1. Fork this repository
2. Create your feature branch (`git checkout -b feature/AmazingResource`)
3. Add new resources under the appropriate category in README.md
4. Ensure you follow this format:
   - `- [Resource Name](link) - Brief description`
   - Description should be concise and clear, no more than one line
   - Links should be directly accessible
5. Commit your changes (`git commit -m 'Add some AmazingResource'`)
6. Push to the branch (`git push origin feature/AmazingResource`)
7. Open a Pull Request

### Resource Addition Guidelines

- **Quality First**: Only add high-quality, useful resources
- **Avoid Duplicates**: Check if the resource already exists
- **Correct Categorization**: Add resources to the most appropriate category
- **Clear Description**: Provide concise but informative descriptions
- **Valid Links**: Ensure all links are accessible

### Category Descriptions

- **Frameworks and Tools**: Development frameworks, SDKs, tool libraries, etc.
- **Models and Datasets**: Pre-trained models, datasets, model weights, etc.
- **Applications and Platforms**: Actual applications, online platforms, services, etc.
- **Research and Papers**: Academic papers, research reports, etc.
- **Community and Resources**: Forums, Discord, Reddit, blogs, etc.
- **Tutorials and Guides**: Tutorials, guides, documentation, etc.

### Commit Message Conventions

- Use clear, descriptive commit messages
- For adding resources, use format: `Add: [Resource Name]`
- For fixes, use format: `Fix: [Issue Description]`
- For updates, use format: `Update: [Update Content]`

## Code of Conduct

- Be friendly and respectful
- Welcome different viewpoints and experiences
- Gracefully accept constructive criticism
- Focus on what's best for the community

## Issue Reporting

If you find issues or have suggestions, please open an Issue.
